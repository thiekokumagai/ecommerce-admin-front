# Plan 04-02: PIX, Pagamentos e Regras Dinâmicas de Descontos e Taxas (Frontend)

## Objective
Implementar as configurações reais de recebimento (PIX, Dinheiro, Débito e Crédito) e criar a tabela interativa com regras dinâmicas de descontos, taxas e parcelamento máximo do cartão.

## Tasks

<task id="1" title="Refatorar Configurações de PIX e Métodos de Recebimento">
<read_first>
- src/components/settings/PaymentSettingsForm.tsx
</read_first>

<action>
1. Refatorar `PaymentSettingsForm.tsx` para carregar valores através da query `useSettings()`.
2. Habilitar os campos de Pix:
   - Switch para ativar/desativar PIX.
   - Select para o **Tipo de Chave** (CPF/CNPJ, E-mail, Telefone).
   - Input para a **Chave Pix** e outro para o **Nome do Titular**.
3. Integrar os switches para pagamentos presenciais na entrega:
   - Dinheiro
   - Cartão de Débito (Maquininha)
   - Cartão de Crédito (Maquininha)
</action>
</task>

<task id="2" title="Criar Formulário de Regras Dinâmicas de Taxas e Descontos">
<read_first>
- src/pages/SettingsPage.tsx
</read_first>

<action>
1. Criar o subcomponente `src/components/settings/PaymentRulesSettingsForm.tsx`.
2. Adicionar o botão primário **"Nova Configuração"** para adicionar linhas dinâmicas de regras no estado local.
3. Cada linha de regra deve renderizar os seguintes controles:
   - **Forma de Pagamento**: Select contendo Pix, Dinheiro, Débito, Crédito (conforme as ativas).
   - **Tipo**: Select contendo Desconto ou Acréscimo.
   - **Valor**: Input numérico formatado com máscara de porcentagem (Ex: `5,00 %`).
   - **Parcelamento Máximo (Cartão de Crédito)**: Select com opções de 1x até 12x (visível apenas se a forma selecionada for Crédito).
   - **Ação de Exclusão**: Botão com ícone de lixeira (`Trash2`) para remover a regra localmente.
</action>
</task>
