# Plan 04-01: Identidade Visual, Contato e Endereço com Busca de CEP (Frontend)

## Objective
Conectar a seção de Configurações Gerais e Endereço da tela `/configuracoes` com os novos endpoints reais da API (`ecommerce-core-api`), adicionando o carrossel de até 7 banners, favicon e busca automatizada por CEP.

## Tasks

<task id="1" title="Configurar Serviços e Hooks do React Query para Settings">
<read_first>
- src/services/product.service.ts (exemplo de padrão de chamadas)
</read_first>

<action>
1. Criar o arquivo `src/services/settings.service.ts` contendo:
   - `getSettings()`: Requisição `GET /settings`
   - `updateSettings(data)`: Requisição `PUT /settings`
   - `uploadMedia(file)`: Requisição `POST /settings/upload` retornando a URL pública.
2. Criar `src/hooks/useSettings.ts`:
   - Expor queries `useSettings` e mutações `useUpdateSettings` com invalidação do cache `"settings"`.
</action>
</task>

<task id="2" title="Implementar Upload de Logo, Favicon e Carrossel de até 7 Banners">
<read_first>
- src/components/settings/GeneralSettingsForm.tsx
</read_first>

<action>
1. Refatorar `GeneralSettingsForm.tsx` para carregar valores através da query `useSettings()`.
2. Adicionar campo de upload para o **Favicon** da loja.
3. Adaptar a seção de **Banner** para suportar a seleção e upload de até 7 imagens promocionais:
   - Exibir uma lista ou carrossel com miniaturas das imagens adicionadas.
   - Permitir deletar banners específicos e ordenar as mídias.
   - Desabilitar a adição quando atingir o limite de 7 banners.
</action>
</task>

<task id="3" title="Integrar Autopreenchimento de Endereço via CEP (ViaCEP)">
<read_first>
- src/components/settings/GeneralSettingsForm.tsx
</read_first>

<action>
1. Adicionar máscara de entrada para o campo **CEP** (`00000-000`).
2. Implementar evento `onChange` que intercepta quando o CEP estiver completo (8 dígitos):
   - Fazer uma busca via fetch no serviço público `https://viacep.com.br/ws/${cep}/json/`.
   - Ao obter sucesso, autopreencher os estados e inputs de: **Logradouro (street)**, **Bairro (neighborhood)**, **Cidade (city)** e **Estado (state)** de forma instantânea.
3. Focar automaticamente no campo **Número** após o preenchimento bem-sucedido para melhorar a usabilidade (UX Premium).
</action>
</task>
