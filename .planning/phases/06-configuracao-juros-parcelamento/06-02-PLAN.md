# Plano 06-02: Tela Interativa de Configuração de Faixas de Juros com Inputs Dinâmicos (Frontend)

Este plano detalha a implementação no painel administrativo (`admin-wonderland`) do formulário dinâmico para cadastrar faixas de juros de parcelamento do cartão de crédito, bem como a correção do bug de exibição do Instagram.

---

## 🎯 Objetivo

* **Inputs Editáveis de Faixas**: Permitir ao lojista configurar faixas como "De Parcela X até Parcela Y com Z% de juros" através de inputs numéricos editáveis.
* **Correção de Bugs**: Garantir que a conta real do Instagram da loja seja carregada e exibida normalmente na tela do painel.
* **Salvamento Unificado**: Persistir tudo de forma transparente com a API do backend local.

---

## 🛠️ Mudanças Propostas

### Frontend (`admin-wonderland`)

#### [MODIFY] [types/settings.ts](file:///c:/sites/podemais-admin/admin-wonderland/src/types/settings.ts)
* Adicionar a propriedade `installmentRules` na tipagem `StoreSettings`:
  ```typescript
  installmentRules: {
    id: string;
    parcelaMin: number;
    parcelaMax: number;
    juros: number;
  }[] | null;
  ```

#### [MODIFY] [GeneralSettingsForm.tsx](file:///c:/sites/podemais-admin/admin-wonderland/src/components/settings/GeneralSettingsForm.tsx)
* **Bug Fix (Instagram oculto)**: Remover a filtragem `"@podemais.cg"` da utilidade `cleanVal` no carregamento de dados do `useEffect`. Usar `setInstagram(settings.instagram || "")` diretamente para assegurar a renderização da conta real da loja.

#### [MODIFY] [PaymentRulesSettingsForm.tsx](file:///c:/sites/podemais-admin/admin-wonderland/src/components/settings/PaymentRulesSettingsForm.tsx)
* Criar uma nova seção visual dentro do formulário intitulada **"Juros de Parcelamento (Cartão de Crédito)"**.
* Criar um estado reativo local `installmentRules` inicializado a partir de `settings.installmentRules || []`.
* Renderizar uma tabela/lista interativa para gerenciar as faixas de parcelamento, onde:
  * O lojista pode clicar em "Adicionar Faixa" para inserir uma nova regra na lista.
  * Cada linha contém inputs numéricos editáveis de:
    - **De Parcela (Min)** (ex: `<Input type="number" value={rule.parcelaMin} />`)
    - **Até Parcela (Max)** (ex: `<Input type="number" value={rule.parcelaMax} />`)
    - **Juros (%)** (ex: `<Input type="number" step="0.01" value={rule.juros} />`)
  * Botão de exclusão (lixeira) para remover a faixa.
* Integrar com o botão "Salvar" de forma que repasse `installmentRules: rules` no payload enviado para o backend.
