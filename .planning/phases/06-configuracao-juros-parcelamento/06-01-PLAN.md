# Plano 06-01: Extensão do Schema, DTO e Repositório da API de Configurações (Backend)

Este plano detalha as alterações no backend (`ecommerce-core-api`) para armazenar e expor as regras de juros customizados por faixas de parcelas para o cartão de crédito.

---

## 🎯 Objetivo

* **Armazenamento de Juros**: Criar o campo `installmentRules` no banco para persistir faixas de juros em formato JSON.
* **Integridade de Tipagem**: Mapear as novas propriedades na entidade `StoreSettings` e validar requisições através do DTO correspondente.

---

## 🛠️ Mudanças Propostas

### Backend (`ecommerce-core-api`)

#### [MODIFY] [schema.prisma](file:///c:/sites/podemais-admin/ecommerce-core-api/prisma/schema.prisma)
* Adicionar o campo `installmentRules` na tabela `StoreSettings`:
  ```prisma
  installmentRules        Json?
  ```

#### [MODIFY] [store-settings.entity.ts](file:///c:/sites/podemais-admin/ecommerce-core-api/src/modules/settings/domain/entities/store-settings.entity.ts)
* Atualizar a interface `StoreSettings` no backend para incluir `installmentRules` (representando o array de faixas de juros).

#### [MODIFY] [get-settings.use-case.ts](file:///c:/sites/podemais-admin/ecommerce-core-api/src/modules/settings/domain/use-cases/get-settings.use-case.ts)
* Inicializar `installmentRules` como um array vazio (`[]`) por padrão quando as configurações da loja forem criadas.

#### [MODIFY] [update-settings.use-case.ts](file:///c:/sites/podemais-admin/ecommerce-core-api/src/modules/settings/domain/use-cases/update-settings.use-case.ts)
* Mapear a propriedade `installmentRules` de entrada no caso de uso de atualização de configurações.

#### [MODIFY] [update-settings.dto.ts](file:///c:/sites/podemais-admin/ecommerce-core-api/src/modules/settings/infrastructure/dtos/update-settings.dto.ts)
* Adicionar a propriedade `installmentRules` no DTO do Swagger com os decorators `@ApiProperty` e `@IsOptional`.

#### [MODIFY] [prisma-settings.repository.ts](file:///c:/sites/podemais-admin/ecommerce-core-api/src/modules/settings/infrastructure/database/prisma-settings.repository.ts)
* Mapear e persistir o campo `installmentRules` na persistência do Prisma (`settings.installmentRules ?? []`).
