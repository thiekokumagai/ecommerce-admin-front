# Plan 24-01: Interface de Criação de Pedido no Admin

## Objective
Implementar a interface de criação manual de pedidos no painel administrativo, contemplando busca de produtos, busca de cliente por telefone, seleção de endereço salvo, aplicação de cupom e escolha de forma de pagamento.

## Tasks

```xml
<task>
  <objective>Criar rota e página principal para Criação de Pedido</objective>
  <read_first>
    - src/routes/index.tsx
    - src/pages/Orders/OrdersPage.tsx
  </read_first>
  <action>
    Criar `CreateOrderPage.tsx` na pasta de Orders.
    Adicionar rota `/pedidos/novo` em `src/routes/index.tsx`.
    A página deve conter a estrutura base (layout) para o fluxo de criação do pedido, dividida em colunas ou seções (Produtos, Cliente, Pagamento, Resumo).
  </action>
  <acceptance_criteria>
    - Acessar `/pedidos/novo` renderiza a nova página vazia/layout básico sem erros.
  </acceptance_criteria>
</task>

<task>
  <objective>Implementar busca e seleção de produtos</objective>
  <read_first>
    - src/pages/Orders/CreateOrderPage.tsx
    - src/services/product.service.ts
  </read_first>
  <action>
    Criar componente `ProductSearch.tsx` que contém um input simples de busca por texto (nome do produto).
    Ao digitar, chamar a API de listagem de produtos.
    Permitir clicar no produto para adicioná-lo à lista do pedido (estado local em `CreateOrderPage`).
    Exibir os produtos selecionados com opção de alterar quantidade ou remover.
  </action>
  <acceptance_criteria>
    - O input de busca encontra produtos e os exibe em um dropdown ou lista.
    - Clicar no produto adiciona-o corretamente ao estado do pedido.
  </acceptance_criteria>
</task>

<task>
  <objective>Implementar busca de cliente por telefone e seleção de endereços</objective>
  <read_first>
    - src/pages/Orders/CreateOrderPage.tsx
    - src/services/customer.service.ts
  </read_first>
  <action>
    Criar componente `CustomerSearch.tsx`.
    Adicionar input com máscara de telefone.
    Ao completar o telefone, buscar o cliente na API (`/customers` ou similar).
    Se encontrado, exibir os dados do cliente e seus endereços salvos (`customer.addresses`) para seleção (radio buttons ou lista clicável).
  </action>
  <acceptance_criteria>
    - Digitar um telefone válido realiza a busca e retorna os dados do cliente.
    - Se houver endereços, eles são exibidos e podem ser selecionados para entrega.
  </acceptance_criteria>
</task>

<task>
  <objective>Adicionar aplicação de cupom e escolha de pagamento</objective>
  <read_first>
    - src/pages/Orders/CreateOrderPage.tsx
  </read_first>
  <action>
    Criar componente `OrderSummary.tsx`.
    Adicionar input para digitar código de cupom de desconto e botão "Aplicar" (validar na API de cupons).
    Adicionar um select ou lista de botões para as Formas de Pagamento (PIX, Dinheiro, Cartão de Crédito/Débito).
    Exibir subtotal (soma dos produtos), desconto do cupom, frete (se aplicável pelo endereço), e Total.
  </action>
  <acceptance_criteria>
    - O cupom inserido é validado e reflete no Total do pedido.
    - O usuário consegue selecionar uma forma de pagamento.
  </acceptance_criteria>
</task>

<task>
  <objective>Finalizar criação do pedido</objective>
  <read_first>
    - src/pages/Orders/CreateOrderPage.tsx
    - src/services/order.service.ts
  </read_first>
  <action>
    Adicionar botão "Finalizar Pedido" em `CreateOrderPage.tsx`.
    Ao clicar, montar o payload contendo: produtos (IDs e quantidades), ID do cliente, ID do endereço selecionado (se houver), cupom aplicado e forma de pagamento.
    Fazer POST para o endpoint de criação de pedidos no painel admin (pode ser o mesmo do cliente ou um específico do admin).
    Ao finalizar com sucesso, redirecionar para a listagem de pedidos ou exibir sucesso.
  </action>
  <acceptance_criteria>
    - O clique no botão envia o payload correto para a API.
    - Sucesso na criação resulta em notificação e redirecionamento apropriado.
  </acceptance_criteria>
</task>
```

## Verification Criteria
- [ ] É possível buscar um produto pelo nome e adicioná-lo à lista.
- [ ] É possível buscar um cliente existente pelo telefone.
- [ ] Ao encontrar o cliente, seus endereços salvos ficam disponíveis para seleção.
- [ ] Pode-se aplicar um cupom de desconto.
- [ ] Ao clicar em "Finalizar Pedido", a requisição é feita corretamente para a API e o pedido é criado.
