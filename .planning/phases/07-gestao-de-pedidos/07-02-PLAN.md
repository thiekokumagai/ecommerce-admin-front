# Plano 07-02: Tela de Listagem de Pedidos Agrupada por Data (Frontend)

Este plano detalha o desenvolvimento da tela de listagem de pedidos no painel administrativo (`ecommerce-admin-front`), integrando-a com o menu de navegação lateral.

---

## 🎯 Objetivo

* **Navegação e Rotas**: Registrar a rota `/pedidos` e mapeá-la na seção de "Vendas" do menu lateral.
* **Interface Reativa**: Exibir listagem agrupada por datas (ex: "Hoje", "18/05/2026") com cartões informando: ID do Pedido, Nome do Cliente, Valor Total, Forma de Pagamento e Status Badge.
* **Filtros e Estado**: Integrar filtros de busca por nome/ID e filtros por status, gerenciando os dados via React Query.

---

## 🛠️ Mudanças Propostas

### 1. Navegação Lateral e Rotas

#### [MODIFY] [AdminSidebar.tsx] (ou arquivo de menu do admin-front)
* Inserir o item "Pedidos" com ícone adequado (e.g. `ShoppingBag`) sob a categoria de "Vendas" direcionando para `/pedidos`.

#### [MODIFY] [App.tsx] (ou arquivo de rotas do admin-front)
* Registrar a rota `/pedidos` acoplada ao componente de página `OrdersPage.tsx`.

---

### 2. Interface de Listagem de Pedidos

#### [NEW] `src/pages/OrdersPage.tsx`
* Criar a página de controle principal:
  * Barra de Ferramentas superior: Input de busca de texto (`id` ou `nome`) e dropdown reativo para filtrar por `status` (Todos, Pendentes, Confirmados, Despachados, Entregues, Cancelados).
  * Painel de Dados: Consome a API do backend através de um hook `useOrders()` (React Query).
  * Agrupador de Datas: Função utilitária local para agrupar o array de pedidos retornado pela data de criação (`createdAt`).
    * Exibe uma barra divisora de data estilizada (e.g. `18/05/2026` ou `Hoje`).
  * Tabela/Lista de Pedidos:
    * Linhas em formato Card contendo:
      * ID Pedido: `#6960`
      * Nome Cliente: `Ana beatriz Martins`
      * Valor: `R$ 167,00`
      * Forma de Pagamento: `PIX` (ou ícone sutil)
      * Status Badge: Estilização por status (e.g., Vermelho para `Cancelado`, Verde para `Concluído`, Amarelo para `Pendente`).
    * Ação de Clique: Clicar em qualquer card de pedido armazena o ID selecionado no estado local (`selectedOrderId`) abrindo o Drawer de Detalhes.
