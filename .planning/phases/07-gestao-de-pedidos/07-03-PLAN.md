# Plano 07-03: Drawer de Detalhes do Pedido e Ação de Cancelamento (Frontend)

Este plano detalha a implementação do Drawer lateral de alta fidelidade para visualização de detalhes de pedidos no painel administrativo (`ecommerce-admin-front`), espelhando precisamente a UI do protótipo enviado.

---

## 🎯 Objetivo

* **Design de Alta Fidelidade**: Desenvolver o componente `OrderDetailDrawer` para renderizar os dados de forma idêntica à captura de tela, respeitando fontes, espaçamentos, ícones de ações rápidas, badge circular de quantidade e resumos financeiros.
* **Fluxo de Cancelamento**: Integrar a mutação com o backend para cancelar o pedido e disparar feedbacks via Sonner Toast.

---

## 🛠️ Mudanças Propostas

### 1. Componente de Detalhes de Pedido (High-Fidelity Drawer)

#### [NEW] `src/components/OrderDetailDrawer.tsx`
* Criar a gaveta lateral (usando `Sheet` ou `Dialog` do Radix/shadcn-ui) recebendo as props `orderId` e `isOpen`.
* **Visual Structure (Header & Actions)**:
  * Título: `< Voltar` à esquerda ou número do pedido `#ID` destacado em roxo/marca.
  * Ações rápidas no canto superior direito:
    * Compartilhar (avião de papel), WhatsApp, Copiar dados, Imprimir, Download PDF, Editar (ícones Lucide em fundo roxo arredondado).
  * Data e Hora: Exibição no padrão brasileiro `DD/MM/AA - HH:MM` (ex: `18/05/26 - 16:23`).
  * Informações do Cliente:
    * Nome: `Ana beatriz Martins` com ícone indicador roxo e seta de expansão.
    * Contato: Link telefônico clicável `(47) 98912-1987`.
* **Visual Structure (Products List)**:
  * Listagem vertical de produtos:
    * Badge de Quantidade: Círculo roxo com fonte branca no canto superior esquerdo da imagem (`1`).
    * Imagem Thumbnail: Miniatura quadrada do produto com bordas arredondadas e fallback de placeholder cinza de alta fidelidade.
    * Título: Nome do produto (ex: `Refil Life Pod 2 Pink Lemonade 10k puffs`) com linhas pontilhadas discretas direcionando para o preço.
    * Preço Unitário: Alinhado à direita (`R$80,00`).
* **Visual Structure (Summary Calculations)**:
  * Sumário de valores formatados:
    * `Total dos itens (X)` -> `R$ 160,00`
    * `Frete` -> `R$ 15,00`
    * `Desconto PIX` -> `-R$ 8,00` (com cor destacada)
    * `Total pedido` -> `R$ 167,00` em negrito
    * `Total recebido` -> `R$ 167,00` em verde destacado
* **Visual Structure (Payment Section)**:
  * Detalhes de faturamento:
    * Pagamento: `Online` (ou Presencial)
    * Forma de pagamento: `PIX` (ou Crédito/Dinheiro)
    * Chave PIX: `(67) 99213-0201`
* **Visual Structure (Shipping Address Section)**:
  * Dados de entrega:
    * Cabeçalho `Para entregar:`
    * Bloco de endereço multilinhas:
      * `Avenida Mato Grosso 3478, 3478`
      * `Santa Fé - diimagem`
      * `Campo Grande - MS`
      * `CEP: 79021-904`
    * Botão de utilidade: `Copiar endereço` que aciona a API de clipboard do navegador mostrando um alerta visual de confirmação.
* **Visual Structure (Footer Buttons)**:
  * Ações finais (alinhadas lado a lado):
    * Botão `Cancelar Pedido`: Estilizado em rosa/vermelho sólido. Aciona a mutação REST `/orders/:id/cancel` via React Query. Mostra loader durante a chamada e atualiza a lista reativamente.
    * Botão `Voltar`: Estilizado em outline com bordas azuis/cinzas, aciona o fechamento do Drawer lateral.
