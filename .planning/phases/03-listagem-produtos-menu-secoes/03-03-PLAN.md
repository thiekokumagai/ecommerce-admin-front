# Plan 03-03: Listagem de produtos (tabela + paginação + bulk)

## Objective
Substituir cards por tabela administrativa com filtros, ações em massa e paginação server-side (30 por página).

## Tasks

<task id="1" title="Estender serviço e tipos para listagem paginada">
<read_first>
- src/services/product.service.ts
- src/types/product.ts
- src/hooks/useProducts.ts
- .planning/phases/03-listagem-produtos-menu-secoes/03-RESEARCH.md
</read_first>

<action>
1. Em `src/types/product.ts`, adicionar:
   - `ProductListParams`: `{ page?: number; limit?: number; search?: string; categoryId?: string }`
   - `ProductListMeta`: `{ page: number; limit: number; hasNextPage: boolean }`
   - Campos em `ProductResponse`: `totalStock: number`, `primarySku: string | null`, `status: 'active' | 'inactive'`

2. Em `product.service.ts`:
   - Alterar `getProducts(params?: ProductListParams)` para `GET /products?page=&limit=&search=&categoryId=`
   - Default `limit: 30`
   - Em `normalizeProduct`, se `item.items` existir: calcular `totalStock` = soma de `stock`, `primarySku` = primeiro item com `sku` ou null, `status` = `totalStock > 0 ? 'active' : 'inactive'`
   - Retornar `{ products: ProductResponse[], meta: ProductListMeta }` onde `hasNextPage = products.length === limit`

3. Em `useProducts.ts`: aceitar params; `queryKey: ['products', params]`; retornar `data.products` e `data.meta`
</action>

<acceptance_criteria>
- `getProducts({ page: 1, limit: 30 })` monta URL com `limit=30`
- `ProductResponse` inclui `totalStock`, `primarySku`, `status`
- `useProducts` invalida cache por params de filtro/página
</acceptance_criteria>
</task>
