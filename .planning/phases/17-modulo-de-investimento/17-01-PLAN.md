# Fase 17: Plano 01 - Criação do Módulo de Investimentos e Controle de Compras

## Objetivos
Criar um sistema de carteira de investimentos apartada do fluxo natural de vendas. O lojista poderá "sangrar" o dinheiro do caixa principal e enviá-lo para este módulo de investimento. Posteriormente, as compras com fornecedores (conferência de estoque) serão debitadas dessa carteira de investimento.

## Etapas do Desenvolvimento (Backend)

1. **Atualização do Prisma Schema (`ecommerce-api/prisma/schema.prisma`)**:
   - Criar o modelo `InvestmentTransaction`.
     ```prisma
     model InvestmentTransaction {
       id          String   @id @default(uuid())
       type        TransactionType // ENTRY (Crédito) | OUTFLOW (Saída/Compra)
       amount      Float
       description String?
       createdAt   DateTime @default(now())
       updatedAt   DateTime @updatedAt
     }
     ```
   - (Nota: O enum `TransactionType` já existe e tem `ENTRY` e `OUTFLOW`).
   - Rodar `npx prisma format` e `npx prisma db push`.

2. **Criação da Camada de Domínio e Repositórios (`ecommerce-api/src/modules/investments`)**:
   - `InvestmentTransaction` Entity.
   - `IInvestmentsRepository` com métodos: `create`, `findAll`, `getSummary` (calcula entradas - saídas).
   - `PrismaInvestmentsRepository` implementando a lógica no banco de dados.

3. **Criação dos Use Cases**:
   - `AddInvestmentUseCase`: Recebe um valor, retira esse valor do Caixa Aberto atual (criando um `CashTransaction` do tipo `OUTFLOW` com a descrição "Transferência para Investimento") e cria um `InvestmentTransaction` do tipo `ENTRY`.
   - `RegisterPurchaseUseCase`: Adiciona um gasto com fornecedores. Cria um `InvestmentTransaction` do tipo `OUTFLOW`.
   - `GetInvestmentSummaryUseCase`: Retorna saldo total (total investido - total gasto), total das entradas e total das saídas.
   - `ListInvestmentTransactionsUseCase`: Retorna o histórico de movimentações com paginação/ordenação.

4. **Controllers e Modules**:
   - `InvestmentsController`: Endpoints `POST /investments/add`, `POST /investments/purchase`, `GET /investments/summary`, `GET /investments/transactions`.
   - `InvestmentsModule`: Registra as injeções de dependência e importa o `CashRegistersModule` se necessário (para usar o caso de uso de saída de caixa).

## Etapas do Desenvolvimento (Frontend)

1. **Serviços e Hooks (`ecommerce-admin-front/src/services/investment.service.ts`)**:
   - Criar arquivo `investment.service.ts` apontando para os endpoints `/investments/*`.
   - Criar as mutations no React Query para adicionar investimento e registrar compra de fornecedor.
   - Criar queries para o histórico e o saldo da carteira.

2. **Criação da Página de Investimentos (`ecommerce-admin-front/src/pages/InvestmentsPage.tsx`)**:
   - Adicionar a rota em `App.tsx` e o atalho no menu lateral (`AdminSidebar.tsx`).
   - Desenvolver um Header com cards de resumo: "Saldo Atual", "Total Investido", "Gasto com Fornecedores". (Permitir que o saldo fique negativo).
   - Tabela de listagem do histórico de transações de investimento (Data, Descrição, Tipo, Valor).
   
3. **Modais de Ação**:
   - **Dialog "Adicionar Investimento"**: Explica ao usuário que o valor será sangrado do caixa vigente. Pede o valor e uma descrição opcional.
   - **Dialog "Registrar Compra / Conferência"**: Pede o valor gasto e a descrição dos produtos/fornecedor.

## Critérios de Aceite (UAT)
- [ ] O banco de dados salva corretamente as transações de investimento.
- [ ] Ao adicionar um investimento, o saldo da carteira aumenta e simultaneamente uma saída (sangria) é lançada no caixa do dia (CashRegister).
- [ ] Ao registrar uma compra, o saldo da carteira diminui corretamente (podendo ficar negativo).
- [ ] A tela de investimentos no admin lista o histórico com clareza e exibe os cards matemáticos atualizados.
