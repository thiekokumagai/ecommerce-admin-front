# Plano 05-01: Filtro Reativo de Placeholders e Dados Mockados (Frontend-Only)

Este plano detalha o refactoring nos formulários do painel administrativo (`admin-wonderland`) para garantir que os campos de texto que recebam dados padrões ou mockados da API carreguem como campos vazios `""`, provendo uma experiência limpa de onboarding.

---

## 🎯 Objetivo

* **Limpar Inputs Automáticos**: Remover chaves PIX provisórias, telefones mockados e o endereço modelo Campo Grande no carregamento dos formulários.
* **Preservar Dados Reais**: Se o usuário preencheu e salvou seus próprios dados reais, eles devem continuar carregando normalmente.

---

## 🛠️ Mudanças Propostas

### [MODIFY] [GeneralSettingsForm.tsx](file:///c:/sites/podemais-admin/admin-wonderland/src/components/settings/GeneralSettingsForm.tsx)
Mapear placeholders de exemplo na inicialização do `useEffect`:
- `storeName`: se for `"Minha Loja"` ou `"Pode Mais"`, setar como `""`.
- `phone`: se for `"(67) 99999-9999"` ou `""`, setar como `""`.
- `instagram`: se for `"@podemais.cg"`, setar como `""`.
- `cep`: se for `"79002-075"`, setar como `""`.
- `street`: se for `"Rua 14 de Julho"`, setar como `""`.
- `number`: se for `"1234"`, setar como `""`.
- `neighborhood`: se for `"Centro"`, setar como `""`.
- `city`: se for `"Campo Grande"`, setar como `""`.
- `state`: se for `"MS"`, setar como `""`.

### [MODIFY] [PaymentSettingsForm.tsx](file:///c:/sites/podemais-admin/admin-wonderland/src/components/settings/PaymentSettingsForm.tsx)
Mapear placeholders de exemplo na inicialização do `useEffect`:
- `pixKey`: se for `"podemais@email.com"`, setar como `""`.
- `pixHolder`: se for `"Pode Mais E-commerce LTDA"`, setar como `""`.
